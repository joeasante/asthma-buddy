    **Step 2: Create `app/views/settings/dose_logs/_form.html.erb`**

    This is the inline dose log form rendered inside each medication card. It submits to `settings_medication_dose_logs_path(medication)` via Turbo Stream:

    ```erb
    <%= form_with model: [medication, dose_log],
          url: settings_medication_dose_logs_path(medication),
          id: "dose_log_form_#{dom_id(medication)}" do |f| %>

      <% if dose_log.errors.any? %>
        <div class="dose-log-errors" role="alert">
          <ul>
            <% dose_log.errors.full_messages.each do |msg| %>
              <li><%= msg %></li>
            <% end %>
          </ul>
        </div>
      <% end %>

      <div class="dose-log-form-fields">
        <%= f.label :puffs, "Puffs taken" %>
        <%= f.number_field :puffs, min: 1, value: dose_log.puffs || medication.standard_dose_puffs, required: true, class: "dose-log-puffs-input" %>

        <%= f.label :recorded_at, "When" %>
        <%= f.datetime_local_field :recorded_at, value: (dose_log.recorded_at || Time.current).strftime("%Y-%m-%dT%H:%M"), class: "dose-log-time-input" %>
      </div>

      <%= f.submit "Log dose", class: "btn-primary" %>
    <% end %>
    ```

    Key decisions:
    - `id: "dose_log_form_#{dom_id(medication)}"` — this is the Turbo Stream target for form reset after create and for replacing on validation error.
    - `value: dose_log.puffs || medication.standard_dose_puffs` — puffs field pre-fills with the medication's standard dose for one-tap convenience.
    - `f.datetime_local_field :recorded_at` with `.strftime("%Y-%m-%dT%H:%M")` — `<input type="datetime-local">` requires this exact format; `Time.current` provides the default.
    - The form url is explicitly set to `settings_medication_dose_logs_path(medication)` because `model: [medication, dose_log]` with a new record would try to build the nested route automatically, but the Settings module namespace means the auto-resolved URL must be explicit. Using explicit `url:` is safer.
    - Do NOT use `form_with url: ..., scope: :dose_log` — use `model:` so Rails generates correct field names (`dose_log[puffs]`, `dose_log[recorded_at]`) matching strong params.

            method: :delete,

    - `turbo_confirm:` uses the custom confirm dialog (same pattern as medication Remove button in Phase 11).

    **Step 1: Update `app/views/settings/medications/_medication.html.erb`**

    Add the dose history section and dose log form to each medication card. The updated partial:

    ```erb
    <%= turbo_frame_tag dom_id(medication) do %>
      <article class="medication-card">
        <div class="medication-card-header">
          <h3 class="medication-card-name"><%= medication.name %></h3>
          <span class="medication-badge medication-badge--<%= medication.medication_type %>">
            <%= medication.medication_type.humanize %>
          </span>
        </div>

        <dl class="medication-card-details">
          <dt>Standard dose</dt>
          <dd><%= medication.standard_dose_puffs %> <%= "puff".pluralize(medication.standard_dose_puffs) %></dd>

          <dt>Starting count</dt>
          <dd><%= medication.starting_dose_count %> doses</dd>

          <% if medication.sick_day_dose_puffs.present? %>
            <dt>Sick-day dose</dt>
            <dd><%= medication.sick_day_dose_puffs %> <%= "puff".pluralize(medication.sick_day_dose_puffs) %></dd>
          <% end %>

          <% if medication.doses_per_day.present? %>
            <dt>Doses per day</dt>
            <dd><%= medication.doses_per_day %></dd>
          <% end %>

          <dt>Remaining</dt>
          <dd id="remaining_count_<%= dom_id(medication) %>"><%= medication.remaining_doses %> doses</dd>
        </dl>

        <section class="dose-log-section" aria-label="Log a dose">
          <h4 class="dose-log-section-title">Log a dose</h4>
          <%= render "settings/dose_logs/form",
                medication: medication,
                dose_log: DoseLog.new(medication: medication, recorded_at: Time.current) %>
        </section>

        <section class="dose-history-section" aria-label="Recent doses">
          <h4 class="dose-history-title">Recent doses</h4>
          <div id="dose_history_<%= dom_id(medication) %>">
            <% recent_logs = medication.dose_logs.sort_by(&:recorded_at).reverse.first(5) %>
            <% if recent_logs.any? %>
              <ul class="dose-history-list">
                <%= render partial: "settings/dose_logs/dose_log", collection: recent_logs, as: :dose_log %>
              </ul>
            <% else %>
              <p class="dose-history-empty">No doses logged yet.</p>
            <% end %>
          </div>
        </section>

        <div class="medication-card-actions">
          <%= link_to "Edit", edit_settings_medication_path(medication), class: "btn-edit" %>
          <%= button_to "Remove", settings_medication_path(medication), method: :delete,
                class: "btn-delete",
                form: { data: { turbo: true, turbo_confirm: "Remove this medication? This can't be undone." } } %>
        </div>
      </article>
    <% end %>
    ```

    Key decisions:
    - `id="remaining_count_#{dom_id(medication)}"` — Turbo Stream target for remaining dose count after create/destroy.
    - `id="dose_history_#{dom_id(medication)}"` — Turbo Stream target for replacing the dose history list.
    - `medication.dose_logs.sort_by(&:recorded_at).reverse.first(5)` — sorts and slices the already-eager-loaded association in Ruby (no additional DB query). The association was loaded via `includes(:dose_logs)` in the controller.
    - `DoseLog.new(medication: medication, recorded_at: Time.current)` — builds a new unsaved DoseLog as the form object. Do NOT use `medication.dose_logs.new(...)` here — that would push the unsaved record into the in-memory association array (MEMORY.md safety rule; same issue as symptom_logs pattern).
    - The `turbo_frame_tag dom_id(medication)` wrapper is preserved from Phase 11 — the inline edit pattern still works unchanged.

    ```erb

      <% if recent_logs.any? %>

      <%= render "settings/dose_logs/form",
            medication: @medication,
            dose_log: DoseLog.new(medication: @medication, recorded_at: Time.current) %>
    <% end %>

    <%= turbo_stream.replace "flash-messages" do %>
      <%= render "layouts/flash" %>
    <% end %>
    ```

    Key decisions:
    - Three `turbo_stream.replace` calls: dose history section, remaining count cell, form reset.
    - `@medication.dose_logs.chronological.limit(5)` — fresh DB query after the save; the association cache may include the new record but `.chronological.limit(5)` guarantees correct order via SQL.
    - Flash via `turbo_stream.replace "flash-messages"` consistent with the project pattern (Phase 11 precedent).
    - Set `flash[:notice] = "Dose logged."` in the controller before `respond_to` so it is available in the `layouts/flash` partial rendered here. The controller `create` action already sets a notice on success — confirm the controller uses `notice:` in the redirect fallback. The turbo_stream path must also set flash directly: add `flash.now[:notice] = "Dose logged."` before `respond_to` in the controller's success branch if not already present.
