Add a constant `LOW_STOCK_DAYS = 14` at the top of the Medication class body (below the associations/validations block, before the scope).

Add a `low_stock?` instance method that returns `true` when days_of_supply_remaining is not nil AND less than LOW_STOCK_DAYS. Return `false` when days_of_supply_remaining is nil (no schedule) — this is the "no division by zero" guard per CONTEXT.md: medications without doses_per_day must never trigger the 14-day warning.

```ruby
LOW_STOCK_DAYS = 14

def low_stock?
  days = days_of_supply_remaining
  days.present? && days < LOW_STOCK_DAYS
end
```

Do NOT add a `low_stock?` check based on raw remaining_doses without considering doses_per_day — reliever inhalers have no schedule and must not trigger the warning per the locked decision in CONTEXT.md.

**Dashboard controller** (`app/controllers/dashboard_controller.rb`):

At the end of `DashboardController#index`, add:
```ruby
@low_stock_medications = user.medications.includes(:dose_logs).select(&:low_stock?)
```
This loads all user medications with their dose_logs (one query, avoids N+1) then filters in Ruby using the `low_stock?` predicate (which calls `days_of_supply_remaining` which calls `remaining_doses` which uses `dose_logs.sum` — but because we used `includes(:dose_logs)`, the sum is computed in Ruby from already-loaded records).
