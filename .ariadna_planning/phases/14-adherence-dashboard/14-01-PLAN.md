Implementation requirements (from CONTEXT.md locked decisions):
- Class method `AdherenceCalculator.call(medication, date)` — single entry point
- Returns a value object with `taken`, `scheduled`, `status` attributes (use `Struct.new(:taken, :scheduled, :status)` or `Data.define` — Rails 8 / Ruby 4 supports both)
- `scheduled` = `medication.doses_per_day` (integer or nil)
- `taken` = count of dose log puffs entries for this medication on the given date (NOT sum of puffs — count of log records). Note: each log event = one administration, so count of records is the correct interpretation for "doses taken". Use `medication.dose_logs.where(recorded_at: date.beginning_of_day..date.end_of_day).count`
- `status` logic:
  - `:no_schedule` if `scheduled.nil?` OR if `date < medication.created_at.to_date` (medication did not exist that day)
  - `:on_track` if `taken >= scheduled` (all scheduled doses logged)
  - `:missed` if `taken < scheduled`

Full implementation skeleton:
```ruby
# frozen_string_literal: true

class AdherenceCalculator
  Result = Struct.new(:taken, :scheduled, :status)

  def self.call(medication, date)
    new(medication, date).call
  end

  def initialize(medication, date)
    @medication = medication
    @date = date
  end

  def call
    scheduled = @medication.doses_per_day
    taken = @medication.dose_logs
                       .where(recorded_at: @date.beginning_of_day..@date.end_of_day)
                       .count

    status = if scheduled.nil? || @date < @medication.created_at.to_date
               :no_schedule
             elsif taken >= scheduled
               :on_track
             else
               :missed
             end

    Result.new(taken, scheduled, status)
  end
end
```
