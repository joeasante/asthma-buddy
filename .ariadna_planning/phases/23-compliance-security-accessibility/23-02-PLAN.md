  <name>Task 1: Differentiate throttle error messages by match name</name>
  <files>config/initializers/rack_attack.rb</files>
  <action>
    Open `config/initializers/rack_attack.rb` and replace the `throttled_responder` lambda.

    IMPORTANT — correct API: `throttled_responder` in rack-attack 6.x receives a `Rack::Attack::Request` object, not a raw env hash. The throttle name that fired is available at `req.env["rack.attack.matched"]` — this is set by `Rack::Attack::Check#matched_by?` and contains the throttle name string (e.g. `"logins/ip"` or `"signups/ip"`).

    Replace:

    ```ruby
    # Custom response for throttled requests
    self.throttled_responder = lambda do |_env|
      [429, { "Content-Type" => "text/plain" }, ["Too many requests. Please wait before trying again."]]
    end
    ```

    With:

    ```ruby
    # Custom response for throttled requests — message varies by which throttle fired
    self.throttled_responder = lambda do |req|
      message = case req.env["rack.attack.matched"]
                when "logins/ip"
                  "Too many sign-in attempts. Please wait 20 seconds before trying again."
                when "signups/ip"
                  "Too many sign-up attempts from this IP address. Please try again later."
                else
                  "Too many requests. Please try again later."
                end
      [429, { "Content-Type" => "text/plain" }, [message]]
    end
    ```

    Do not change anything else in the file — throttle rules, cache store, or the test-environment disable line must remain exactly as they are.
  </action>

  </verify>
