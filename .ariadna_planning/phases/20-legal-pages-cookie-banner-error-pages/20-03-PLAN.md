<objective>
Create branded 404 and 500 error pages that match the app's visual design, wire them into Rails' exceptions_app mechanism so real errors are served by our controller (not the default Rails fallback pages), and produce a standalone `public/maintenance.html` that renders correctly without any Rails dependency.

Purpose: Users seeing the Rails white screen or plaintext error page during incidents damages trust in a health app. Branded error pages maintain the app's visual identity even during failures. ERR-01 and ERR-02 are required for public launch.
Output: ErrorsController, two error views, errors.css, public/maintenance.html, routes, application.rb config, controller tests.
</objective>

1. Create `app/controllers/errors_controller.rb`:
   ```ruby
   # frozen_string_literal: true

   class ErrorsController < ApplicationController
     allow_unauthenticated_access

     def not_found
       render status: :not_found
     end

     def internal_server_error
       render status: :internal_server_error
     end
   end
   ```

2. Add error routes in `config/routes.rb`. These MUST be placed at the very END of the routes file, just before the final `end`, so they catch only what nothing else matched:
   ```ruby
   match "/404", to: "errors#not_found",              via: :all
   match "/500", to: "errors#internal_server_error",  via: :all
   ```

3. Add `config.exceptions_app = self.routes` to `config/application.rb` inside the `class Application < Rails::Application` block, alongside the existing config statements:
   ```ruby
   config.exceptions_app = self.routes
   ```
