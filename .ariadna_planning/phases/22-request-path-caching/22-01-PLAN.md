<objective>
Cache the unread notification badge count in Solid Cache so the `notifications.unread.count` query is not executed on every authenticated page load.

Purpose: The badge count fires on every authenticated request via `ApplicationController#set_notification_badge_count`. At current scale the query is fast, but it is unbounded — one DB hit per page load per user regardless of whether anything changed. Solid Cache is already running; this plan wires the existing infrastructure to eliminate the redundant reads.

Output: `ApplicationController#set_notification_badge_count` reads from cache. `Notification` model invalidates that cache key on create and on marking-as-read. Tests verify cache is written, read, and invalidated correctly.
</objective>

In `app/controllers/application_controller.rb`, update `set_notification_badge_count` to wrap the DB query in `Rails.cache.fetch`:

```ruby
def set_notification_badge_count
  user_id = Current.user.id
  @unread_notification_count = Rails.cache.fetch(
    "unread_notifications/#{user_id}",
    expires_in: 1.hour
  ) do
    Current.user.notifications.unread.count
  end
end
```

In `app/models/notification.rb`, add two private invalidation methods and wire them to commit callbacks:

```ruby
after_create_commit  :invalidate_badge_cache
after_update_commit  :invalidate_badge_cache, if: :saved_change_to_read?

private

  def invalidate_badge_cache
    Rails.cache.delete("unread_notifications/#{user_id}")
  end
```

The `user_id` column is already present on the notifications table (belongs_to :user). Use `user_id` (the FK integer) directly — no association load needed inside the callback. Place the private method at the end of the existing private section, after `create_low_stock_for`.

No other files need changing. `NotificationsController`, `ErrorsController`, and `CookieNoticesController` all use `skip_before_action :set_notification_badge_count` — they never call this method and need no changes.
