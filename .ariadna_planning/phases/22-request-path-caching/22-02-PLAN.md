<objective>
Cache the three dashboard aggregate variables (`@preventer_adherence`, `@reliever_medications`, `@active_illness`) in Solid Cache so the multi-query `set_dashboard_vars` is not re-executed on every dashboard load and every Turbo Stream response after dose log actions.

Purpose: `set_dashboard_vars` runs several queries including an `includes(:dose_logs)` association load and `AdherenceCalculator.call` per preventer medication. This fires on every dashboard load AND on every Turbo Stream settings response (via `Settings::BaseController`). Caching the result with write-triggered invalidation eliminates redundant reads without any consistency risk.

Output: `DashboardVariables#set_dashboard_vars` stores and retrieves the full result hash from cache. `DoseLog` and `HealthEvent` models delete the cache key on write operations. Cache key includes `Date.current` for natural midnight rotation. Tests verify cache population, cache reads, and invalidation from both models.
</objective>

**`app/controllers/concerns/dashboard_variables.rb`** â€” wrap the body of `set_dashboard_vars` in a `Rails.cache.fetch` call:

```ruby
def set_dashboard_vars
  user  = Current.user
  today = Date.current

  cached = Rails.cache.fetch("dashboard_vars/#{user.id}/#{today}", expires_in: 5.minutes) do
    preventer_adherence = user.medications
      .where(medication_type: :preventer)
      .where(course: false)
      .includes(:dose_logs)
      .select { |m| m.doses_per_day.present? }
      .map { |m| { medication: m, result: AdherenceCalculator.call(m, today) } }
    reliever_medications = user.medications
      .where(medication_type: :reliever)
      .where(course: false)
      .includes(:dose_logs)
      .chronological
      .to_a
    active_illness = user.health_events
      .where(event_type: :illness)
      .where(ended_at: nil)
      .order(recorded_at: :desc)
      .first

    { preventer_adherence: preventer_adherence, reliever_medications: reliever_medications, active_illness: active_illness }
  end

  @preventer_adherence  = cached[:preventer_adherence]
  @reliever_medications = cached[:reliever_medications]
  @active_illness       = cached[:active_illness]
end
```

The cache stores a plain Ruby Hash. Rails marshals ActiveRecord objects and the `AdherenceCalculator::Result` Struct correctly. The `Date.current` segment in the key ensures the cache is naturally invalid after midnight without any callback â€” yesterday's adherence result is never served for today.
