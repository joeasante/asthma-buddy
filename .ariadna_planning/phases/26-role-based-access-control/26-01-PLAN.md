  <action>
    1. Add `gem "pundit"` to Gemfile and run `bundle install`.

    2. Create migration `replace_admin_boolean_with_role_enum`:
       - Add `role` integer column with default 0 (member), null: false
       - Backfill: `UPDATE users SET role = 1 WHERE admin = true` (1 = admin)
       - Remove `admin` column
       - Add index on `role` column

    3. Update User model:
       - Add `enum :role, { member: 0, admin: 1 }, default: :member`
       - The enum provides `admin?`, `member?` methods which replaces the old boolean `admin?` — existing code calling `user.admin?` continues to work transparently.

    4. Run `bin/rails db:migrate`.

    5. Create `app/policies/application_policy.rb` (deny-by-default):
       - Default all actions to `false` (deny)
       - Provide `Scope` class that defaults to `scope.none`
       - Add helper method `admin?` that delegates to `user.admin?`

    6. Create individual policies:

       **SymptomLogPolicy**: All CRUD actions allowed for the record owner (`record.user == user`). Scope: `scope.where(user: user)`.
       **PeakFlowReadingPolicy**: Same owner-based pattern as SymptomLogPolicy.
       **HealthEventPolicy**: Same owner-based pattern.
       **MedicationPolicy**: Same owner-based pattern.
       **DoseLogPolicy**: Owner check via `record.medication.user == user`. Scope: `scope.joins(:medication).where(medications: { user_id: user.id })`.
       **NotificationPolicy**: Owner-based (`record.user == user`). `index?` and `mark_read?` and `mark_all_read?` allowed for owner.
       **ProfilePolicy**: `show?` and `update?` always true (users manage their own profile — the controller already scopes to Current.user).
       **UserPolicy**: `index?` admin only. `toggle_admin?` admin only (and record != user AND `User.admin.count > 1` when demoting — last-admin protection). `update_role?` admin only (and record != user). Scope: admin sees all users, members see none.
       **DashboardPolicy**: `index?` always true (all authenticated users can see their own dashboard).

    7. Update `test/fixtures/users.yml`: Replace `admin: true` with `role: 1` (or `role: admin`) for admin_user fixture. Remove any `admin: false` lines from other fixtures (default is member).
  </action>
  <verify>
    - `bin/rails db:migrate` succeeds
    - `bin/rails runner "puts User.first.role"` outputs "member" or "admin"
    - `bin/rails runner "puts User.roles"` outputs the enum hash
    - `bin/rails runner "u = User.new; puts u.admin?"` outputs false (default is member)
    - `ls app/policies/` shows all policy files
  </verify>
  <done>
    User model uses role enum (member/admin). Pundit gem installed. ApplicationPolicy denies by default. Individual policies exist for every resource controller. Old `admin?` method still works via enum.
  </done>

    1. In `ApplicationController`:
       - Add `include Pundit::Authorization`
       - Add `after_action :verify_authorized, unless: :skip_authorization?`
       - Add `after_action :verify_policy_scoped, only: :index, unless: :skip_authorization?`
       - Add `rescue_from Pundit::NotAuthorizedError, with: :user_not_authorized`
       - Implement `user_not_authorized`: for HTML, redirect back with fallback to root and flash alert "You are not authorized to perform this action."; for JSON, render `{ error: "Forbidden" }` with status 403.
       - Implement `skip_authorization?` private method that returns true for controllers that don't need Pundit (unauthenticated controllers). The cleanest approach: define a class method `skip_pundit` in ApplicationController that sets an inheritable accessor, then call it in controllers that handle unauthenticated access.

    2. Controllers that skip Pundit entirely (unauthenticated access):
       - `SessionsController` — login/logout, no resource to authorize
       - `RegistrationsController` — signup, no authenticated user
       - `EmailVerificationsController` — email links, no authenticated user
       - `PasswordsController` — password reset, no authenticated user
       - `HomeController` — public landing page
       - `PagesController` — public static pages (privacy, terms, cookies)
       - `CookieNoticesController` — dismiss cookie banner
       - `ErrorsController` — error pages, no auth context
       - `CspReportsController` — inherits from ActionController::Base (not ApplicationController), so Pundit is not included at all; no changes needed
       - `Settings::BaseController` — no actions of its own (just sets layout/concerns); skip Pundit here so child controllers handle their own authorization

       For each of these (except CspReportsController), add `skip_pundit` (or `skip_after_action :verify_authorized` and `skip_after_action :verify_policy_scoped, raise: false`). These controllers handle unauthenticated access or have no actions to authorize.

       NOTE: If a `Test::SessionsController` or similar test-only controller exists, add it to the skip list as well.

    3. Run `bin/rails test` to verify no Pundit::AuthorizationNotPerformedError is raised by unauthenticated controllers.

    - `bin/rails test test/controllers/registrations_controller_test.rb` passes (no Pundit errors on signup)

       **Settings::AccountsController**: `authorize :account, :destroy?` — all authenticated users can delete their own account.

       **PreventerHistoryController**: `authorize :preventer_history, :index?` — all authenticated users.

       **RelieverUsageController**: `authorize :reliever_usage, :index?` — all authenticated users.
