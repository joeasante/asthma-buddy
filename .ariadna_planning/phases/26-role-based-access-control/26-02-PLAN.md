       - `toggle_registration` action: calls `SiteSetting.toggle_registration!`, `authorize :site_setting, :toggle_registration?`, redirects back to admin dashboard with notice "Registration is now #{SiteSetting.registration_open? ? 'open' : 'closed'}."

    5. Update `config/routes.rb` — inside the `namespace :admin` block:
       - Add `post "site_settings/toggle_registration", to: "site_settings#toggle_registration", as: :toggle_registration`

    6. Update `ApplicationController`:
       - Replace the ENV-based `registration_open?` method with `SiteSetting.registration_open?`
       - Remove `allowed_emails` and `allowed_email?` methods (the ENV-based ALLOWED_EMAILS approach is replaced by the database-backed toggle). If `allowed_email?` is used elsewhere (check SessionsController), keep it but make `registration_open?` use SiteSetting.
       - Keep `registration_open?` as a helper_method.

       IMPORTANT: Check if `allowed_email?` is used in SessionsController for login restriction. If it IS used for login (not just registration), keep the ALLOWED_EMAILS logic for login but use SiteSetting for registration. The RBAC-04 requirement is specifically about registration toggle, not login restriction.

    7. Update `app/views/admin/dashboard/index.html.erb`:
       - Add a "Site Settings" section (or card) showing the current registration status (Open/Closed) with a toggle button.
       - The toggle button uses `button_to` with `admin_toggle_registration_path`, method: :post.
       - Show a green "Open" badge or red "Closed" badge for current status.
       - Include turbo_confirm: "Are you sure you want to #{SiteSetting.registration_open? ? 'close' : 'open'} registration?"

    8. Update `app/views/registrations/new.html.erb`:
       - Wrap the form in a conditional: if `registration_open?` is false, show a centered message "Registration is currently closed" with a link back to the sign-in page, and do NOT render the form.
       - If `registration_open?` is true, show the form as normal.

    9. Update `RegistrationsController#require_registration_open`:
       - This already calls `registration_open?` which now delegates to SiteSetting. No change needed unless the redirect message needs updating.
       - Verify the controller redirects with "Registration is currently closed." message (it already does).

    10. Run `bin/rails db:migrate` and verify.
