<objective>
Record user activity on every login and notify the admin when a new user registers.

Purpose: Zero visibility into user activity is a blind spot for a live app. This plan instruments the two highest-value signals — login frequency/recency and new signups — with no external dependencies.
Output: Migration adding last_sign_in_at + sign_in_count to users table; SessionsController updating those columns on login; AdminMailer sending signup notification to the configured admin_email credential; User model after_create_commit callback; tests covering all three.
</objective>

    In `app/controllers/sessions_controller.rb`, inside `#create`, after `start_new_session_for user` and before the `respond_to` block, add:
    ```ruby
    user.update_columns(
      last_sign_in_at: Time.current,
      sign_in_count:   user.sign_in_count + 1
    )
    ```

    Use `update_columns` (bypasses validations and callbacks, single SQL UPDATE — correct for tracking metadata). The variable is `user`, not `Current.user`, because Current is not yet set at this point in the request cycle — `start_new_session_for` just called, but the controller's `Current.user` accessor resolves via the session cookie which hasn't been read back yet. Using the local `user` variable is safe.

    class AdminMailer < ApplicationMailer
      def new_signup(user)
