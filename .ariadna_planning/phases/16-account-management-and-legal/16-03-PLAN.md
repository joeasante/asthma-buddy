Add `before_action :set_cookie_notice_flag` to ApplicationController. This sets `@show_cookie_notice` based on the session flag. Place it after the existing `before_action` calls.

```ruby
before_action :set_cookie_notice_flag

private

def set_cookie_notice_flag
  @show_cookie_notice = !session[:cookie_notice_shown]
end
```

Keep all existing `private` methods. The new `set_cookie_notice_flag` method joins the existing `set_no_store_cache_for_authenticated_users` private method.

**app/controllers/cookie_notices_controller.rb — create:**

```ruby
# frozen_string_literal: true

class CookieNoticesController < ApplicationController
  allow_unauthenticated_access

  def dismiss
    session[:cookie_notice_shown] = true
    head :no_content
  end
end
```

Using `head :no_content` (204) because the dismiss button is a Turbo form submission. The Stimulus controller will hide the banner on success via a CSS class toggle — no redirect or Turbo Stream needed.

**config/routes.rb — add route:**

Add inside `Rails.application.routes.draw` block, near the other resource routes:

The `cookie-notice` Stimulus controller handles the client-side hide animation. Create `app/javascript/controllers/cookie_notice_controller.js`:

```javascript
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  dismiss() {
    this.element.classList.add("cookie-notice--dismissed")
    // Remove after CSS transition completes
    this.element.addEventListener("transitionend", () => {
      this.element.remove()
    }, { once: true })
  }
}
```

Register in `app/javascript/controllers/index.js` — add:
```javascript
import CookieNoticeController from "./cookie_notice_controller"
application.register("cookie-notice", CookieNoticeController)
```
