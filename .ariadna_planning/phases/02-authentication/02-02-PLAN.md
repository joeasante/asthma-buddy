    1. Add a signed token method to User model for email verification:
       - Add `generates_token_for :email_verification, expires_in: 24.hours` (Rails 7.1+ token API)
       - This gives us `user.generate_token_for(:email_verification)` and `User.find_by_token_for(:email_verification, token)`
       - No extra columns needed — the token is stateless and signed

    2. Create app/mailers/user_mailer.rb:
       - Method `email_verification(user)` that:
         - Sets `@user = user`
         - Sets `@verification_url` using the signed token: `email_verification_url(token: @user.generate_token_for(:email_verification))`
         - Sends to `user.email` with subject "Verify your email address — Asthma Buddy"

    3. Create email views:
       - app/views/user_mailer/email_verification.html.erb: "Hi, please verify your email by clicking the link below." with `link_to "Verify my email", @verification_url`. Mention the link expires in 24 hours.
       - app/views/user_mailer/email_verification.text.erb: Plain text version with the URL.

    4. Update RegistrationsController#create:
       - After successfully saving the user, call `UserMailer.email_verification(@user).deliver_later`
       - Keep the redirect to login page with the existing notice message

    5. Create app/controllers/email_verifications_controller.rb:
       - `show` action: finds user via `User.find_by_token_for(:email_verification, params[:token])`
       - If user found and not yet verified: set `user.update!(email_verified_at: Time.current)`, redirect to login with notice "Email verified! You can now sign in."
       - If user found and already verified: redirect to login with notice "Email already verified."
       - If token invalid/expired: redirect to login with alert "Invalid or expired verification link."

    6. Update config/routes.rb:
       - Add `resource :email_verification, only: [:show], param: :token`
       - The show route becomes GET /email_verification/:token

    7. Gate login on email verification in SessionsController#create:
       - After authenticating the user (User.authenticate_by), check `user.email_verified_at.present?`
       - If NOT verified: do NOT create session. Redirect back to login with alert "Please verify your email address before signing in. Check your inbox for a verification link."
       - If verified: proceed with normal session creation

    8. Configure Action Mailer for development:
       - In config/environments/development.rb, ensure `config.action_mailer.default_url_options = { host: "localhost", port: 3000 }` is set
       - Ensure `config.action_mailer.delivery_method = :test` or `:letter_opener` if available (`:test` is fine — emails appear in ActionMailer::Base.deliveries in tests)

    IMPORTANT: Use `generates_token_for` (Rails 7.1+) — no need for a separate token column. The token is derived from the user's attributes and is self-expiring.

    IMPORTANT: Keep the password "password123" convention from 02-01 fixtures for test compatibility.
  </action>

    - Login attempt with unverified user is rejected with flash message
    - After setting email_verified_at, login succeeds
