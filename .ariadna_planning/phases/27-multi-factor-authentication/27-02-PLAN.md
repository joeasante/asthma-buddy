**2. Create `app/controllers/mfa_challenge_controller.rb`:**

- Inherits from `ApplicationController`.
- `skip_pundit` (unauthenticated flow — same pattern as SessionsController).
- `allow_unauthenticated_access` (user is not yet authenticated).
- `rate_limit to: 5, within: 1.minute, only: :create` with a response handler that redirects to `new_mfa_challenge_path` with alert "Too many attempts. Try again later." for HTML and returns 429 JSON.
- `before_action :require_pending_mfa` — private method that redirects to `new_session_path` unless `session[:pending_mfa_user_id]` is present. Also check expiry: if `session[:pending_mfa_at]` is missing or older than 5 minutes (`Time.current.to_i - session[:pending_mfa_at].to_i > 300`), clear both session keys and redirect to `new_session_path` with alert "Your verification session expired. Please sign in again."

- `new` action: renders the TOTP entry form. No special logic needed.

- `create` action (per locked decision — pending MFA session state, don't authenticate before TOTP verification):
  1. Find user: `User.find_by(id: session[:pending_mfa_user_id])`. If nil, redirect to `new_session_path`.
  2. Try `user.verify_otp(params[:otp_code])`. If true:
     - Clear `session[:pending_mfa_user_id]` and `session[:pending_mfa_at]`.
     - Call `start_new_session_for user` (from Authentication concern).
     - Set `session[:last_seen_at] = Time.current`.
     - Update sign-in tracking: `user.update_columns(last_sign_in_at: Time.current)` and `User.where(id: user.id).update_all("sign_in_count = sign_in_count + 1")`.
     - Redirect to `after_authentication_url`.
  3. Else try `user.verify_recovery_code(params[:otp_code])`. If true:
     - Same session/auth flow as above.
     - Redirect with notice: "Recovery code used. You have #{user.recovery_codes_remaining} remaining."
  4. Else: `flash.now[:alert] = "Invalid code. Please try again."`, `render :new, status: :unprocessable_entity`.

  Support `respond_to` with `format.json` returning appropriate JSON responses (201 on success, 401 on invalid code).

**3. Modify `app/controllers/sessions_controller.rb`:**

- In `create` action: after the existing `email_verified_at?` check and before `start_new_session_for`, add the MFA interception:

  ```ruby
  if user.otp_required_for_login?

    session[:pending_mfa_at] = Time.current.to_i
    return respond_to do |format|
      format.html { redirect_to new_mfa_challenge_path }
      format.json { render json: { error: "MFA required", mfa_required: true }, status: :forbidden }
    end
  end
  ```
  This MUST go before `start_new_session_for` — the locked decision says "don't authenticate before TOTP verification."

- In `create` action: also clear stale pending MFA at the very start of create (before authenticate_by), same as `new`:
  ```ruby
  session.delete(:pending_mfa_user_id)
  session.delete(:pending_mfa_at)
  ```

**4. Create `app/controllers/settings/security_controller.rb`:**

- Inherits from `Settings::BaseController`.
- All actions use `authorize :settings, :show?` (reuse existing SettingsPolicy — security is user-scoped, no special policy needed per research open question #3).

- `show` action: renders security settings page showing MFA status (`Current.user.otp_required_for_login?`).

- `setup` action: generates a new secret, stores in `session[:pending_otp_secret]` (NEVER write to DB before confirmation — anti-pattern from research). Creates provisioning URI and QR SVG:
  ```ruby
  session[:pending_otp_secret] = ROTP::Base32.random
  totp = ROTP::TOTP.new(session[:pending_otp_secret], issuer: "Asthma Buddy")
  @provisioning_uri = totp.provisioning_uri(Current.user.email_address)
  @qr_svg = helpers.mfa_qr_svg(@provisioning_uri)
  @manual_key = session[:pending_otp_secret]
  ```

- `confirm_setup` action (POST): verifies the code against the pending secret:
  ```ruby
  secret = session[:pending_otp_secret]
  return redirect_to settings_security_path, alert: "Setup expired. Please try again." unless secret

  totp = ROTP::TOTP.new(secret, issuer: "Asthma Buddy")
  if totp.verify(params[:otp_code].to_s, drift_behind: 15)
    @recovery_codes = Current.user.enable_mfa!(secret)
    session.delete(:pending_otp_secret)
    redirect_to recovery_codes_settings_security_path
  else
    flash.now[:alert] = "Invalid code. Scan the QR code and try again."
    @provisioning_uri = totp.provisioning_uri(Current.user.email_address)
    @qr_svg = helpers.mfa_qr_svg(@provisioning_uri)
    @manual_key = secret
    render :setup, status: :unprocessable_entity
  end
  ```

- `recovery_codes` action (GET): `@recovery_codes = Current.user.recovery_codes`. Renders the recovery codes page.

- `download_recovery_codes` action (POST): generates a text file download:
  ```ruby
  codes = Current.user.recovery_codes
  content = "Asthma Buddy Recovery Codes\n"
  content += "Generated: #{Time.current.strftime('%Y-%m-%d')}\n"
  content += "Each code can only be used once.\n\n"
  content += codes.map.with_index(1) { |code, i| "#{i.to_s.rjust(2)}. #{code}" }.join("\n")
  send_data content, filename: "asthma-buddy-recovery-codes.txt", type: "text/plain"
  ```

- `disable` action (GET): renders the password re-authentication form.

- `confirm_disable` action (POST): verifies password, then disables MFA:
  ```ruby
  unless Current.user.authenticate(params[:password])
    flash.now[:alert] = "Incorrect password."
    render :disable, status: :unprocessable_entity
    return
  end
  Current.user.disable_mfa!
  redirect_to settings_security_path, notice: "Two-factor authentication has been disabled."
  ```

- `regenerate_recovery_codes` action (GET): renders password re-auth form for regeneration.

- `confirm_regenerate_recovery_codes` action (POST): verifies password, generates new codes:
  ```ruby
  unless Current.user.authenticate(params[:password])
    flash.now[:alert] = "Incorrect password."
    render :regenerate_recovery_codes, status: :unprocessable_entity
    return
  end
  @recovery_codes = Current.user.regenerate_recovery_codes!
  redirect_to recovery_codes_settings_security_path, notice: "New recovery codes generated. Your old codes are no longer valid."
  ```
  This requires adding `regenerate_recovery_codes!` to the User model — generate 10 new codes, save to `otp_recovery_codes`, return codes array.
