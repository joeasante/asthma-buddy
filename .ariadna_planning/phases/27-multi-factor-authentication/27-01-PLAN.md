<task type="auto">
  <name>Task 2: User model MFA methods and unit tests</name>
  <files>app/models/user.rb, test/models/user_test.rb, test/fixtures/users.yml</files>
  <action>
1. Add MFA methods to `app/models/user.rb`:

   Add encrypted attribute declarations (per locked decision — use Rails AR Encryption):
   ```ruby
   encrypts :otp_secret, deterministic: false
   encrypts :otp_recovery_codes, deterministic: false
   ```

   Add these instance methods:

   - `otp_required_for_login?` — returns true only if `otp_secret.present? && otp_required_for_login` (the boolean column). This guards against edge cases where one is set without the other.

   - `verify_otp(code)` — creates `ROTP::TOTP.new(otp_secret, issuer: "Asthma Buddy")`, calls `totp.verify(code.to_s, drift_behind: 15, after: last_otp_at.to_i)`. If result is truthy (returns timestamp integer), update `last_otp_at` to `Time.at(result)` and return true. Otherwise return false. Return false early if `otp_secret.blank?` or `code.blank?`.

   - `verify_recovery_code(code)` — normalize code: `code.to_s.strip.downcase.delete("-")`. Find the code in `recovery_codes_array`. If found, remove it (delete_at the index), save the remaining codes as comma-separated string to `otp_recovery_codes`, return true. If not found, return false. Return false early if `code.blank?`.

   - `enable_mfa!(secret)` — generate 10 recovery codes via `SecureRandom.hex(5)`, update user with: `otp_secret: secret, otp_required_for_login: true, otp_recovery_codes: codes.join(","), last_otp_at: nil`. Return the codes array.

   - `disable_mfa!` — update user with: `otp_secret: nil, otp_required_for_login: false, otp_recovery_codes: nil, last_otp_at: nil`.

   - `recovery_codes` — returns `recovery_codes_array` (public accessor).

   - `recovery_codes_remaining` — returns `recovery_codes_array.size`.

   Private helper:
   - `recovery_codes_array` — `(otp_recovery_codes || "").split(",").reject(&:blank?)`
   - `generate_recovery_codes` — `10.times.map { SecureRandom.hex(5) }`

2. Add a fixture user with MFA enabled in `test/fixtures/users.yml`:
   Add `mfa_user` fixture. NOTE: Since otp_secret is encrypted via AR Encryption, the fixture value must be a plaintext string that AR Encryption will handle — fixtures with `encrypts` columns work with plaintext values in YAML (Rails encrypts on load). Set a known ROTP Base32 secret like `JBSWY3DPEHPK3PXP`, `otp_required_for_login: true`, and `otp_recovery_codes` with a few known codes (e.g., `"abc123def4,1234567890"`). Ensure the fixture has `email_address`, `password_digest` (via ERB BCrypt), `email_verified_at`, and `role: member`.

3. Add model tests in `test/models/user_test.rb`:

   - `test "otp_required_for_login? returns false when MFA not enabled"` — verified_user fixture, assert_not.
   - `test "otp_required_for_login? returns true when MFA enabled"` — mfa_user fixture, assert.
   - `test "enable_mfa! sets otp_secret and returns 10 recovery codes"` — call on verified_user with a generated secret, assert otp_required_for_login? is true, assert 10 codes returned, assert otp_secret is set.
   - `test "verify_otp accepts valid TOTP code"` — enable MFA on a user with a known secret, generate a valid code via `ROTP::TOTP.new(secret).now`, assert verify_otp returns true.
   - `test "verify_otp rejects invalid code"` — assert verify_otp("000000") returns false.
   - `test "verify_otp prevents replay"` — verify a code, then verify the same code again immediately, assert second call returns false (last_otp_at blocks it).
   - `test "verify_otp returns false for blank code"` — assert verify_otp("") returns false, assert verify_otp(nil) returns false.
   - `test "verify_recovery_code consumes code on use"` — enable MFA, get codes, verify first code, assert true, assert recovery_codes_remaining is 9, verify same code again, assert false.
   - `test "verify_recovery_code normalizes input"` — enable MFA, get codes, verify with uppercase and dashes added, assert true.
   - `test "disable_mfa! clears all MFA fields"` — enable MFA, then disable, assert otp_secret nil, otp_required_for_login false, recovery_codes empty.
   - `test "otp_secret is encrypted at rest"` — enable MFA with a known secret, read raw database value via `User.connection.select_value("SELECT otp_secret FROM users WHERE id = #{user.id}")`, assert it does NOT equal the plaintext secret.
