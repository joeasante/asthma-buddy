  <name>Task 1: Migration + User model API key concern</name>
  <files>
    db/migrate/YYYYMMDDHHMMSS_add_api_key_columns_to_users.rb
    app/models/concerns/api_authenticatable.rb
    app/models/user.rb
    test/models/user_api_key_test.rb

  <action>
    1. Create migration adding two columns to users table:
       - `api_key_digest` (string, nullable, indexed uniquely) — stores SHA-256 hex digest
       - `api_key_created_at` (datetime, nullable) — when the key was generated
       Add a unique index on api_key_digest (for fast lookup during authentication).

    2. Create `app/models/concerns/api_authenticatable.rb` with:
       - `generate_api_key!` — generates a secure random token (SecureRandom.hex(32) = 64-char hex string), computes SHA-256 digest via `Digest::SHA256.hexdigest(token)`, saves digest and timestamp to user, returns the plaintext token (caller shows it once).
       - `revoke_api_key!` — sets api_key_digest and api_key_created_at to nil.
       - `api_key_active?` — returns true if api_key_digest is present.
       - Class method `self.authenticate_by_api_key(token)` — computes SHA-256 of token, finds user by digest. Returns nil if token blank or no match.

    3. Include `ApiAuthenticatable` in `User` model.

    4. Create `test/models/user_api_key_test.rb` with tests:
       - `generate_api_key!` returns a 64-char hex string
       - `generate_api_key!` stores a non-nil digest different from the plaintext
       - `generate_api_key!` sets api_key_created_at
       - Calling `generate_api_key!` again replaces the previous key
       - `revoke_api_key!` clears digest and created_at
       - `User.authenticate_by_api_key` with valid token returns the user
       - `User.authenticate_by_api_key` with invalid token returns nil
       - `User.authenticate_by_api_key` with nil/blank returns nil
       - `User.authenticate_by_api_key` after revocation returns nil
       - `api_key_active?` returns true when key exists, false after revocation

    Run `bin/rails db:migrate` after creating migration.
  </action>
  <verify>
    `bin/rails db:migrate` succeeds.
    `bin/rails test test/models/user_api_key_test.rb` — all tests pass.
  </verify>
  <done>
    User model has generate_api_key!, revoke_api_key!, api_key_active?, and User.authenticate_by_api_key class method. All model tests pass.
  </done>
