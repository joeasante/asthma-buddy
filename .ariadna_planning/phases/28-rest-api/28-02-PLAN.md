  <action>
    1. Add API routes to config/routes.rb (BEFORE the root route):
       ```ruby
       namespace :api do
         namespace :v1 do
           resources :symptom_logs, only: [:index]
           resources :peak_flow_readings, only: [:index]
           resources :medications, only: [:index]
           resources :dose_logs, only: [:index]
           resources :health_events, only: [:index]
         end
       end
       ```

    2. Create `app/controllers/api/v1/base_controller.rb`:
       - Inherit from `ActionController::API` (NOT ApplicationController — API controllers should be lightweight, no session, no CSRF, no browser checks)
       - Include `Pundit::Authorization`
       - `after_action :verify_authorized` (Pundit enforcement)
       - Define `pundit_user` returning `Current.user`
       - `before_action :authenticate_api_key!`
       - Private `authenticate_api_key!`:
         - Extract token from `request.headers["Authorization"]` matching `Bearer <token>`
         - Call `User.authenticate_by_api_key(token)` — if nil, render 401 error
         - If found, set `Current.user = user` (so Pundit and scoping work)
       - Private pagination helper `paginate(scope)`:
         - Parse `params[:page]` (default 1, min 1) and `params[:per_page]` (default 25, min 1, max 100)
         - Compute total count, offset, apply `.limit().offset()` to scope
         - Return `{ records:, meta: { page:, per_page:, total: } }`
       - Private `date_filter(scope, date_column: :recorded_at)`:
         - If `params[:date_from]` present, add `.where("#{date_column} >= ?", Date.parse(params[:date_from]).beginning_of_day)`
         - If `params[:date_to]` present, add `.where("#{date_column} <= ?", Date.parse(params[:date_to]).end_of_day)`
         - Rescue `Date::Error` and render 400 error for invalid dates
         - Return filtered scope
       - Private `render_error(status:, message:, details: nil)`:
         - `render json: { error: { status: status, message: message, details: details }.compact }, status: status`
       - `rescue_from Pundit::NotAuthorizedError` rendering 403 via render_error
       - `rescue_from ActiveRecord::RecordNotFound` rendering 404 via render_error

    3. Create 5 resource controllers, each inheriting from `Api::V1::BaseController`:

       **Api::V1::SymptomLogsController#index:**
       - `authorize SymptomLog` (uses existing SymptomLogPolicy or create one if missing — should scope to current user)
       - `scope = policy_scope(SymptomLog).order(recorded_at: :desc)`
       - Apply date_filter (date_column: :recorded_at)
       - Paginate and render: each log as `{ id, symptom_type, severity, triggers, recorded_at, created_at }`
       - Response: `{ data: [...], meta: { page, per_page, total } }`

       **Api::V1::PeakFlowReadingsController#index:**
       - `authorize PeakFlowReading`
       - `scope = policy_scope(PeakFlowReading).order(recorded_at: :desc)`
       - Apply date_filter (date_column: :recorded_at)
       - Paginate and render: `{ id, value, zone, time_of_day, recorded_at, created_at }`

       **Api::V1::MedicationsController#index:**
       - `authorize Medication`
       - `scope = policy_scope(Medication).order(created_at: :desc)`
       - Paginate and render: `{ id, name, medication_type, dose_unit, standard_dose_puffs, doses_per_day, starting_dose_count, remaining_doses, created_at }`
       - Use `.includes(:dose_logs)` to avoid N+1 on remaining_doses if it depends on dose_logs count

       **Api::V1::DoseLogsController#index:**
       - `authorize DoseLog`
       - `scope = policy_scope(DoseLog).includes(:medication).order(recorded_at: :desc)`
       - Apply date_filter (date_column: :recorded_at)
       - Paginate and render: `{ id, medication_id, medication_name (from association), puffs, recorded_at, created_at }`

       **Api::V1::HealthEventsController#index:**
       - `authorize HealthEvent`
       - `scope = policy_scope(HealthEvent).order(occurred_on: :desc)`
       - Apply date_filter (date_column: :occurred_on)
       - Paginate and render: `{ id, event_type, title, occurred_on, created_at }`

       For serialization, use plain `render json:` with `.as_json(only: [...])` or build hashes manually. Do NOT use Jbuilder for API controllers (unnecessary overhead for simple JSON). Build response hashes inline in each controller for clarity.

    4. Check that Pundit policies exist for all 5 models. The existing web controllers already use Pundit, so policies should exist. Each policy's `Scope` should resolve to `user.model_name_plural` (e.g., `user.symptom_logs`). If any policy scope is missing, add it. The `index?` action should return `true` for authenticated users (members and admins can access their own data).
