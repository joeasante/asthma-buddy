Configure SQLite to run in WAL (Write-Ahead Logging) mode across all environments so the database supports the concurrent reads required by later phases.

Purpose: WAL mode is a locked decision from the project constraints. It allows readers and writers to operate concurrently without blocking each other — essential for a multi-user app on SQLite.
Output: A Rails initializer that sets `PRAGMA journal_mode=WAL` on every connection, and confirmation that schema load succeeds.
</objective>

    1. Create `config/initializers/database_wal.rb` with an `ActiveSupport::Notifications` subscriber on `"!connection.active_record"` — or use the Rails 8 SQLite3 adapter `database.yml` property. For Rails 8 with the sqlite3 gem >= 2.1, the cleanest approach is to add `properties: { journal_mode: wal }` under the `default:` section of `config/database.yml`. This sets WAL mode through the adapter's connection properties without a separate initializer.

    Preferred approach (Rails 8 / sqlite3 gem >= 2.1):
    In `config/database.yml`, under the `default: &default` section, add:
    ```yaml
    properties:
      journal_mode: wal
    ```

    If the sqlite3 gem version does not support the `properties` key, fall back to an initializer approach:
    Create `config/initializers/database_wal.rb`:
    ```ruby
    # frozen_string_literal: true

    # Enable WAL (Write-Ahead Logging) mode for SQLite across all environments.
    # WAL mode allows concurrent reads and writes without blocking — essential for
    # multi-user access on SQLite.
    ActiveSupport.on_load(:active_record) do
      ActiveRecord::ConnectionAdapters::SQLite3Adapter.prepend(
        Module.new do
          def configure_connection
            super
            execute("PRAGMA journal_mode=WAL;")
          end
        end
      )
    end
    ```

    Verify which approach works by checking the sqlite3 gem version in Gemfile.lock and the adapter's supported options.

    2. Run `bin/rails db:create` and `bin/rails db:schema:load` to confirm no errors.
    3. Confirm WAL mode is active: `bin/rails runner "puts ActiveRecord::Base.connection.execute('PRAGMA journal_mode').first.inspect"` — output must include `wal`.
