**Controller — app/controllers/symptom_logs_controller.rb:**

```ruby
# frozen_string_literal: true
class SymptomLogsController < ApplicationController
  def index
    @symptom_log = Current.user.symptom_logs.new(recorded_at: Time.current)
    @symptom_logs = Current.user.symptom_logs.chronological.includes(:rich_text_notes)
  end

  def create
    @symptom_log = Current.user.symptom_logs.new(symptom_log_params)

    if @symptom_log.save
      respond_to do |format|
        format.turbo_stream
        format.html { redirect_to symptom_logs_path, notice: "Symptom logged." }
      end
    else
      respond_to do |format|
        format.turbo_stream { render turbo_stream: turbo_stream.replace("symptom_log_form", partial: "form", locals: { symptom_log: @symptom_log }), status: :unprocessable_entity }
        format.html { render :index, status: :unprocessable_entity }
      end
    end
  end

  private

  def symptom_log_params
    params.require(:symptom_log).permit(:symptom_type, :severity, :recorded_at, :notes)
  end
end
```

Key implementation decisions:
- `Current.user.symptom_logs.new(...)` — all queries are user-scoped via the association. Never `SymptomLog.find()` or `SymptomLog.all`.
- `includes(:rich_text_notes)` — prevents N+1 when rendering notes for each entry in the list. ActionText stores notes in `action_text_rich_texts`; the preload association name is `:rich_text_notes` (Rails convention for `has_rich_text :notes`).
- `format.turbo_stream` without a block on success renders `create.turbo_stream.erb` automatically.
- On validation failure, status 422 (`:unprocessable_entity`) is required for Turbo to process the response as an error. Without 422, Turbo Drive treats the response as a redirect and ignores the stream.
- `recorded_at` is pre-populated with `Time.current` in the index action — so the form shows the current time as the default timestamp.

**Turbo Stream response — app/views/symptom_logs/create.turbo_stream.erb:**

```erb
<%= turbo_stream.prepend "symptom_logs_list", partial: "symptom_log", locals: { symptom_log: @symptom_log } %>
<%= turbo_stream.replace "symptom_log_form" do %>
  <%= render "form", symptom_log: Current.user.symptom_logs.new(recorded_at: Time.current) %>
<% end %>
```

Two stream actions on success:
1. `prepend` — adds the new entry at the top of the `symptom_logs_list` turbo frame (newest first, matching chronological desc order)
2. `replace` — replaces the form with a fresh blank form (clearing all fields including the Trix editor)

The form's turbo frame DOM id is `symptom_log_form`. The list's DOM id is `symptom_logs_list`. These ids are set in the views (Task 2).

```erb
    <%= form.datetime_local_field :recorded_at, required: true %>

    <%= form.label :notes, "Notes (optional)" %>
    <%= form.rich_text_area :notes, aria: { label: "Notes" } %>
  </div>

  <%= form.submit "Log symptom" %>
```

Implementation notes:
- `form_with model: symptom_log, url: symptom_logs_path` — always POSTs to create (no update route in Phase 3).
- `SymptomLog.symptom_types.keys.map { |k| [k.humanize.gsub("_", " "), k] }` — converts enum keys to human-readable labels (e.g., "shortness_of_breath" → "Shortness of breath") while storing the string key as the value.
- `include_blank:` on selects ensures the user actively makes a choice (blank is not valid).
- `datetime_local_field :recorded_at` renders an HTML5 `<input type="datetime-local">` — browsers provide a native date/time picker. The controller pre-fills with `Time.current`.
- `form.rich_text_area :notes` — ActionText's form helper that renders the Trix editor.
- Error display uses `role="alert" aria-live="assertive"` for screen reader announcement.

```ruby
# frozen_string_literal: true
require "test_helper"

class SymptomLogsControllerTest < ActionDispatch::IntegrationTest
  setup do
    @user = users(:verified_user)
    @other_user = users(:unverified_user)
    post session_url, params: { email_address: @user.email_address, password: "password" }
  end

  # --- INDEX ---

  test "index redirects unauthenticated user to sign in" do
    delete session_url
    get symptom_logs_url
    assert_redirected_to new_session_url
  end

  test "index renders for authenticated user" do
    get symptom_logs_url
    assert_response :success
    assert_select "h1", "Log a Symptom"
  end

  test "index shows only current user's symptom logs" do
    # alice's entry from fixtures
    get symptom_logs_url
    assert_response :success
    assert_select "##{dom_id(symptom_logs(:alice_wheezing))}"
    # bob's entry must NOT appear
    assert_select "##{dom_id(symptom_logs(:bob_coughing))}", count: 0
  end

  # --- CREATE ---

  test "create saves a valid symptom log and responds with turbo stream" do
    assert_difference "SymptomLog.count", 1 do
      post symptom_logs_url,
        params: { symptom_log: { symptom_type: "coughing", severity: "moderate", recorded_at: Time.current } },
        headers: { "Accept" => "text/vnd.turbo-stream.html" }
    end
    assert_response :success
    assert_equal "text/vnd.turbo-stream.html", response.media_type
  end

  test "create scopes new entry to current user" do
    post symptom_logs_url,
      params: { symptom_log: { symptom_type: "wheezing", severity: "mild", recorded_at: Time.current } }
    assert_equal @user, SymptomLog.last.user
  end

  test "create with invalid params renders unprocessable_entity" do
    assert_no_difference "SymptomLog.count" do
      post symptom_logs_url,
        params: { symptom_log: { symptom_type: "", severity: "", recorded_at: "" } },
        headers: { "Accept" => "text/vnd.turbo-stream.html" }
    end
    assert_response :unprocessable_entity
  end

  test "create without authentication redirects to sign in" do
    delete session_url
    post symptom_logs_url,
      params: { symptom_log: { symptom_type: "coughing", severity: "mild", recorded_at: Time.current } }
    assert_redirected_to new_session_url
  end
end
```

Implementation notes:
- `setup` signs in via POST to session_url using the `verified_user` fixture credentials (password: "password" as set in Phase 2 fixtures via `BCrypt::Password.create('password')`).
- The isolation test uses `dom_id(symptom_logs(:alice_wheezing))` to check the specific fixture entry appears, then verifies bob's entry does not. This directly tests the SYMP-01 multi-user isolation requirement.
- Turbo Stream requests use `Accept: text/vnd.turbo-stream.html` header to trigger the `format.turbo_stream` branch.
- Status 422 on validation failure is tested — critical for Turbo to correctly handle the error response.
