    Create `app/views/settings/medications/` directory and the following two files.

    **index.html.erb** — the medication list page:
    ```erb
    <% content_for :title, "Medications — Settings" %>

    <h1>Medications</h1>

    <section aria-label="Your medications">
      <div id="medications_list">
        <% if @medications.any? %>
          <%= render @medications %>
        <% else %>
          <div class="medications-empty-state" id="medications_empty_state">
            <p>You have not added any medications yet.</p>
            <%= link_to "Add your first medication", new_settings_medication_path, class: "btn-primary" %>
          </div>
        <% end %>
      </div>
    </section>

    <div class="medications-actions">
      <%= link_to "Add medication", new_settings_medication_path, class: "btn-primary" %>
    </div>
    ```

    Note: `render @medications` uses Rails' collection rendering — it renders `_medication.html.erb` for each record. The `id="medications_list"` div is the Turbo Stream target for create/destroy.

    **_medication.html.erb** — card partial with Turbo Frame for inline editing:
    ```erb
    <%= turbo_frame_tag dom_id(medication) do %>
      <article class="medication-card">
        <div class="medication-card-header">
          <h3 class="medication-card-name"><%= medication.name %></h3>
          <span class="medication-badge medication-badge--<%= medication.medication_type %>">
            <%= medication.medication_type.humanize %>
          </span>
        </div>

        <dl class="medication-card-details">
          <dt>Standard dose</dt>
          <dd><%= medication.standard_dose_puffs %> <%= "puff".pluralize(medication.standard_dose_puffs) %></dd>

          <dt>Starting count</dt>
          <dd><%= medication.starting_dose_count %> doses</dd>

          <% if medication.sick_day_dose_puffs.present? %>
            <dt>Sick-day dose</dt>
            <dd><%= medication.sick_day_dose_puffs %> <%= "puff".pluralize(medication.sick_day_dose_puffs) %></dd>
          <% end %>

          <% if medication.doses_per_day.present? %>
            <dt>Doses per day</dt>
            <dd><%= medication.doses_per_day %></dd>
          <% end %>
        </dl>

        <div class="medication-card-actions">
          <%= link_to "Edit", edit_settings_medication_path(medication), class: "btn-edit" %>
          <%= button_to "Remove", settings_medication_path(medication), method: :delete,
                class: "btn-delete",
                form: { data: { turbo: true, turbo_confirm: "Remove this medication? This can't be undone." } } %>
        </div>
      </article>
    <% end %>
    ```

    The entire card is wrapped in `turbo_frame_tag dom_id(medication)` so the Edit link loads the edit form into the same frame. The `dom_id(medication)` for a Medication with id=5 produces `"medication_5"`.

    Do NOT add a link_to for a show/detail page — there is no show action (deferred per CONTEXT.md).

    **_form.html.erb** — shared form partial used by both new and edit views:
    ```erb
    <%= form_with model: [:settings, medication], id: "medication_form", data: { turbo: true } do |form| %>
      <% if medication.errors.any? %>
        <div role="alert" aria-live="assertive">
          <p><%= pluralize(medication.errors.count, "error") %> prevented this medication from being saved:</p>
          <ul>
            <% medication.errors.each do |error| %>
              <li><%= error.full_message %></li>
            <% end %>
          </ul>
        </div>
      <% end %>

      <div class="field">
        <%= form.label :name, "Medication name" %>
        <%= form.text_field :name, placeholder: "e.g. Ventolin, Clenil Modulite", required: true %>
      </div>

      <div class="field">
        <%= form.label :medication_type, "Type" %>
        <%= form.select :medication_type,
              Medication.medication_types.keys.map { |k| [k.humanize, k] },
              { include_blank: "Select a type", selected: medication.medication_type },
              required: true %>
      </div>

      <div class="field">
        <%= form.label :standard_dose_puffs, "Standard dose (puffs)" %>
        <%= form.number_field :standard_dose_puffs, min: 1, step: 1, required: true %>
      </div>

      <div class="field">
        <%= form.label :starting_dose_count, "Starting dose count" %>
        <p class="field-hint">How many doses are in the inhaler when you start tracking.</p>
        <%= form.number_field :starting_dose_count, min: 0, step: 1, required: true %>
      </div>

      <fieldset class="field-group">
        <legend>Optional</legend>

        <div class="field">
          <%= form.label :sick_day_dose_puffs, "Sick-day dose (puffs)" %>
          <p class="field-hint">Leave blank if not applicable.</p>
          <%= form.number_field :sick_day_dose_puffs, min: 1, step: 1 %>
        </div>

        <div class="field">
          <%= form.label :doses_per_day, "Doses per day" %>
          <p class="field-hint">Required for preventer adherence tracking. Leave blank for relievers.</p>
          <%= form.number_field :doses_per_day, min: 1, step: 1 %>
        </div>
      </fieldset>

      <%= form.submit medication.persisted? ? "Update medication" : "Add medication" %>

      <% if medication.persisted? %>
        <%= link_to "Cancel", settings_medications_path,
              data: { turbo_frame: dom_id(medication) },
              class: "btn-cancel" %>
      <% end %>
    <% end %>
    ```

    The Cancel link on the edit form uses `data: { turbo_frame: dom_id(medication) }` — clicking it loads the show partial back into the frame, restoring the card without a full reload.

    **edit.html.erb** — rendered into the Turbo Frame, so only the form content (no full layout duplication):
    ```erb
    <%= turbo_frame_tag dom_id(@medication) do %>
      <%= render "form", medication: @medication %>
    <% end %>
    ```

    The edit.html.erb wraps the form in the same `turbo_frame_tag dom_id(@medication)` that the card uses — Turbo replaces the card content with the form.

    **create.turbo_stream.erb** — successful create:
    ```erb
    <%= turbo_stream.prepend "medications_list", partial: "settings/medications/medication", locals: { medication: @medication } %>
    <%= turbo_stream.replace "medication_form" do %>
      <%= render "settings/medications/form", medication: Current.user.medications.new %>
    <% end %>
    <%= turbo_stream.replace "flash-messages" do %>
      <div id="flash-messages">
        <p role="status" class="flash flash--notice">Medication added.</p>
      </div>
    <% end %>
    ```

    Note: replaces the form with a fresh blank form after successful create (consistent with symptom_logs pattern). Also removes the empty-state element if present via the prepend to `medications_list`.

    **update.turbo_stream.erb** — successful update (replaces the card's frame with updated content):
    ```erb
    <%= turbo_stream.replace dom_id(@medication), partial: "settings/medications/medication", locals: { medication: @medication } %>
    <%= turbo_stream.replace "flash-messages" do %>
      <div id="flash-messages">
        <p role="status" class="flash flash--notice">Medication updated.</p>
      </div>
    <% end %>
    ```

    **destroy.turbo_stream.erb** — removes the card:
    ```erb
    <%= turbo_stream.remove dom_id(@medication) %>
    <%= turbo_stream.replace "flash-messages" do %>
      <div id="flash-messages">
        <p role="status" class="flash flash--notice">Medication removed.</p>
      </div>
    <% end %>
    ```

    Flash replace uses `id="flash-messages"` — consistent with the project-wide flash convention (STATE.md: "use turbo_stream.replace 'flash-messages' for non-accumulating flash").
