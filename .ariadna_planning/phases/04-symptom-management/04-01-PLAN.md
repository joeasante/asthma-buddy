**app/controllers/symptom_logs_controller.rb** — Add a private `set_symptom_log` before_action scoped to the current user, then implement three new actions:

```ruby
before_action :set_symptom_log, only: %i[ edit update destroy ]

def edit
  # @symptom_log set by before_action — renders edit.html.erb implicitly
  # but since we use Turbo Frame inline editing, the frame targets the partial
  # No explicit render needed; Turbo Frame replaces the frame content with edit form
end

def update
  if @symptom_log.update(symptom_log_params)
    respond_to do |format|
      format.turbo_stream
      format.html { redirect_to symptom_logs_path, notice: "Symptom updated." }
    end
  else
    respond_to do |format|
      format.turbo_stream { render turbo_stream: turbo_stream.replace(dom_id(@symptom_log), partial: "symptom_logs/edit_form", locals: { symptom_log: @symptom_log }), status: :unprocessable_entity }
      format.html { render :edit, status: :unprocessable_entity }
    end
  end
end

def destroy
  @symptom_log.destroy
  respond_to do |format|
    format.turbo_stream { render turbo_stream: turbo_stream.remove(dom_id(@symptom_log)) }
    format.html { redirect_to symptom_logs_path, notice: "Symptom deleted." }
  end
end

private

def set_symptom_log
  @symptom_log = Current.user.symptom_logs.find(params[:id])
  # RecordNotFound raised automatically if ID belongs to another user -> 404
end
```

The `symptom_log_params` method already exists and permits all fields — no changes needed there.

**app/views/symptom_logs/_symptom_log.html.erb** — Wrap the entire article in a `turbo_frame_tag` keyed to `dom_id(symptom_log)`. Add Edit and Delete action buttons inside the frame. The Edit link targets the same frame so clicking it replaces the entry with the edit form inline. The Delete button uses a form with `data-turbo-confirm`.

```erb
<%= turbo_frame_tag dom_id(symptom_log) do %>
  <article class="symptom-log-entry">
    <dl>
      <dt>Symptom</dt>
      <dd><%= symptom_log.symptom_type.humanize.gsub("_", " ").capitalize %></dd>

      <dt>Severity</dt>
      <dd><%= symptom_log.severity.capitalize %></dd>

      <dt>When</dt>
      <dd><time datetime="<%= symptom_log.recorded_at.iso8601 %>">
        <%= symptom_log.recorded_at.strftime("%e %b %Y at %H:%M") %>
      </time></dd>

      <% if symptom_log.notes.present? %>
        <dt>Notes</dt>
        <dd><%= symptom_log.notes %></dd>
      <% end %>
    </dl>

    <div class="symptom-log-actions">
      <%= link_to "Edit", edit_symptom_log_path(symptom_log), class: "btn-edit" %>
      <%= button_to "Delete", symptom_log_path(symptom_log), method: :delete,
            data: { turbo_confirm: "Delete this entry?" },
            class: "btn-delete",
            form: { data: { turbo: true } } %>
    </div>
  </article>
<% end %>
```

Key points:
- `turbo_frame_tag dom_id(symptom_log)` — the frame id matches what the Turbo Stream replace targets
- Edit link has no `data-turbo-frame` override needed — it defaults to the enclosing frame, so clicking it loads the edit response into this same frame inline
- Delete uses `button_to` with `method: :delete` — Rails generates a small form internally
- `data: { turbo_confirm: "Delete this entry?" }` triggers the browser-native confirm dialog before submission
- The outer `id` attribute on `<article>` is removed (the turbo_frame_tag now provides the id)
