The chart window is already defined as `week_start = Date.current.beginning_of_week(:monday)` with `recent_readings` covering that range. Derive the chart window end as `Date.current`.

Add this block after `@chart_data` is built (before `@low_stock_medications`):

```ruby
# Health event markers for the 7-day chart — one entry per event in window.
# Chart window matches recent_readings: week_start..Date.current (Mon–today).
chart_start = week_start
chart_end   = Date.current

@health_event_markers = user.health_events
  .where(recorded_at: chart_start.beginning_of_day..chart_end.end_of_day)
  .order(recorded_at: :asc)
  .map do |e|
    {
      date:         e.recorded_at.to_date.to_s,   # "YYYY-MM-DD"
      type:         e.event_type,
      label:        event_marker_label(e),
      css_modifier: e.event_type_css_modifier
    }
  end
```

Add a private helper `event_marker_label` to `DashboardController`:

```ruby
private

MARKER_LABELS = {
  "hospital_visit"    => "Hosp",
  "gp_appointment"    => "GP",
  "illness"           => "Ill",
  "medication_change" => "Rx",
  "other"             => "Evt"
}.freeze

def event_marker_label(event)
  MARKER_LABELS[event.event_type] || "Evt"
end
```

Add a helper function (outside the class, near the top alongside `cssVar`, `toRgba`, etc.) that returns a colour for a given `css_modifier` string. Use CSS custom properties where available, falling back to hardcoded values:

```javascript
// Returns a stroke colour for a health event marker by its css_modifier.
// Uses hardcoded fallbacks — event colours are not in CSS vars.
function eventMarkerColor(cssModifier) {
  const map = {
    "hospital-visit":    "#dc2626",  // red
    "gp-appointment":    "#2563eb",  // blue
    "illness":           "#d97706",  // amber
    "medication-change": "#7c3aed",  // purple
    "other":             "#64748b"   // slate
  }
  return map[cssModifier] || "#64748b"
}
```

In `renderPeakFlowBandsChart()`, register an inline Chart.js `afterDraw` plugin that draws vertical markers. Add this **before** the `new Chart(...)` call:

```javascript
const healthEvents = this.healthEventsValue || []

// Build a lookup from date string ("YYYY-MM-DD") to chart x-axis label ("Day D")
// so we can map event dates to x positions on the chart.
const dateLabelMap = {}
data.forEach(d => { dateLabelMap[d.date] = toDayLabel(d.date) })

const markerPlugin = {
  id: "healthEventMarkers",
  afterDraw(chart) {
    if (!healthEvents.length) return

    const ctx    = chart.ctx
    const xAxis  = chart.scales.x
    const yAxis  = chart.scales.y
    const top    = yAxis.top
    const bottom = yAxis.bottom

    healthEvents.forEach(event => {
      const label = dateLabelMap[event.date]
      if (!label) return  // event date not in current chart window

      const xPos = xAxis.getPixelForValue(label)
      if (xPos === undefined || isNaN(xPos)) return

      const color = eventMarkerColor(event.css_modifier)

      ctx.save()

      // Vertical dashed line
      ctx.beginPath()
      ctx.setLineDash([4, 3])
      ctx.strokeStyle = color
      ctx.lineWidth   = 1.5
      ctx.globalAlpha = 0.75
      ctx.moveTo(xPos, top)
      ctx.lineTo(xPos, bottom)
      ctx.stroke()

      // Short label text at top of line
      ctx.setLineDash([])
      ctx.globalAlpha = 1
      ctx.fillStyle   = color
      ctx.font        = "bold 10px system-ui, sans-serif"
      ctx.textAlign   = "center"
      ctx.fillText(event.label, xPos, top - 4)

      ctx.restore()
    })
  }
}
```

Pass the plugin into the chart options:

```javascript
this.chart = new Chart(this.element, {
  type: "line",
  data: { labels, datasets },
  options: { ... },
  plugins: [markerPlugin]   // ADD plugins array here
})
```

The existing `renderPeakFlowBandsChart` already builds `this.chart = new Chart(...)`. Modify that call to add `plugins: [markerPlugin]` as a top-level key alongside `type`, `data`, and `options`.

**Implementation constraint:** `xAxis.getPixelForValue` in Chart.js 4.x accepts the label string (as stored in `labels`). `dateLabelMap` translates ISO date to `"Sat 7"` format matching what `toDayLabel` produces for chart labels — this ensures correct x-position lookup.
