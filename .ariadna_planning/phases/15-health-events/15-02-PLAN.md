**Delete event:**

The health events delete button uses `data-turbo-confirm` which triggers the custom `confirm_controller.js` dialog, NOT the browser native confirm. Use `find("dialog.confirm-dialog button[data-action='confirm#accept']").click`.

```ruby
test "user can delete an event and it disappears from the list" do
  event = health_events(:alice_medication_change)
```

**Implementation notes:**
- `datetime-local` fields accept `"YYYY-MM-DDTHH:MM"` format in Capybara's `fill_in`
- Fixture `alice_gp_appointment` is point-in-time (no ended_at); `alice_illness_ongoing` has no ended_at; `alice_illness_resolved` has ended_at set
- The end-date Stimulus controller hides the duration section for point-in-time event types — system tests that need to set ended_at must first select a duration type
- Use `wait:` argument on assertions that follow Turbo mutations (default wait is usually sufficient; add `wait: 3` only if needed)
- `dom_id(event)` produces e.g. `health_event_1` — matches `turbo_frame_tag dom_id(health_event)` in `_event_row.html.erb`
