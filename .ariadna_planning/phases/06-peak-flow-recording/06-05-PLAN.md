Fix three view-layer bugs together because they interact (the flash replace requires the layout wrapper to exist before the stream response uses it).

**_form.html.erb** — Add `required: true` to the number_field :value call. The field currently has `min: 1, max: 900` but no required attribute, so a blank submission passes browser constraint validation and the request reaches the server where Rails casts nil — but the blank-submit path does not reliably surface the error via Turbo. Adding `required: true` stops the blank form at the browser before any XHR fires, giving immediate inline feedback consistent with how the over-900 validation already works.

Change this line:
```
<%= form.number_field :value,
      min: 1, max: 900,
      placeholder: "e.g. 430",
      class: "peak-flow-value-input",
      aria: { label: "Peak flow reading in litres per minute" } %>
```
To:
```
<%= form.number_field :value,
      min: 1, max: 900,
      required: true,
      placeholder: "e.g. 430",
      class: "peak-flow-value-input",
      aria: { label: "Peak flow reading in litres per minute" } %>
```

**application.html.erb** — Wrap the existing flash notices in a `<div id="flash-messages">` so the Turbo Stream replace has a stable DOM target. The existing flash `<p>` tags become children of this div. The div is always rendered (even when no flash is present) so the target id exists on every page load.

Replace this block inside `<main id="main-content">`:
```erb
<% if notice.present? %>
  <p role="status" class="flash flash--notice"><%= notice %></p>
<% end %>
<% if alert.present? %>
  <p role="alert" class="flash flash--alert"><%= alert %></p>
<% end %>
```
With:

<div id="flash-messages">

**create.turbo_stream.erb** — Two fixes:

1. The form replace block must wrap the rendered partial in `turbo_frame_tag` with the matching id `"peak_flow_reading_form"`. Without this wrapper the `<turbo-frame>` element disappears from the DOM after the first replace, breaking subsequent submissions. The partial itself does not contain the outer turbo-frame tag — the stream response must supply it.

2. Switch from `turbo_stream.prepend "main-content"` to `turbo_stream.replace "flash-messages"` so each new flash replaces the previous one instead of accumulating. Use `raw()` around `@flash_message` so the HTML span added in Task 2 renders correctly (not escaped).

Replace the entire file with:
```erb
<%# Reset the form — wrap partial in turbo_frame_tag so the frame element persists in DOM %>
<%= turbo_stream.replace "peak_flow_reading_form" do %>
  <%= turbo_frame_tag "peak_flow_reading_form" do %>
    <%= render "form",
          peak_flow_reading: Current.user.peak_flow_readings.new(
            recorded_at: Time.current.change(sec: 0)
          ),
          has_personal_best: @has_personal_best %>
  <% end %>
<% end %>

<%# Replace flash container — not prepend — so messages don't stack %>
<%= turbo_stream.replace "flash-messages" do %>
  <div id="flash-messages">
    <p role="status" class="flash flash--notice"><%= raw @flash_message %></p>
  </div>
<% end %>
```

**peak_flow_readings_controller.rb** — Update `zone_flash_message` to wrap the zone name (and percentage) in a `<span>` with a CSS class that carries the colour. Mark the return value `html_safe` so Rails does not escape the HTML when it is rendered via `raw()` in the stream template (already applied in Task 1).

Zone-to-CSS-class mapping (aligns with `--severity-*` custom properties established in symptom_timeline.css):
- `"green"` → `zone-label--green`
- `"yellow"` → `zone-label--yellow`
- `"red"` → `zone-label--red`

Replace the `zone_flash_message` private method with:

```ruby
def zone_flash_message(reading)
  if reading.zone.nil?
    "Reading saved \u2014 set your personal best to see your zone."
  else
    zone_name = reading.zone.capitalize
    pct       = reading.zone_percentage
    coloured  = "<span class=\"zone-label zone-label--#{reading.zone}\">#{zone_name} Zone (#{pct}% of personal best)</span>"
    "Reading saved \u2014 #{coloured}.".html_safe
  end
end
```

Note: The em-dash is the unicode escape `\u2014` to avoid encoding issues in the source file. Use a literal em dash (—) if the file already uses one consistently.
