**1b. app/controllers/reliever_usage_controller.rb**

Create `RelieverUsageController < ApplicationController` with a single `index` action. The controller is scoped to `Current.user` throughout — never expose another user's data.

Period param: accept `8` or `12` (weeks). Default 8. Reject any other value and fall back to 8.

```ruby
@weeks = params[:weeks].to_i.in?([8, 12]) ? params[:weeks].to_i : 8
```

Build date range as the past N complete weeks plus the current partial week:

```ruby
period_start = @weeks.weeks.ago.to_date
date_range   = period_start..Date.current
```

Query all reliever-type medications for the current user:

```ruby
@relievers = Current.user.medications
  .where(medication_type: :reliever)
  .chronological
```

If `@relievers.empty?`, set `@weekly_data = []` and `@correlation = nil` and return early (the view renders the "no relievers" empty state).

Batch-load all dose logs for reliever medications in the period in one query (same pattern as AdherenceController — avoids N+1):

```ruby
all_logs = Current.user.dose_logs
  .where(medication: @relievers)
  .where(recorded_at: date_range.first.beginning_of_day..date_range.last.end_of_day)
```

Check whether any logs exist:

```ruby
@has_logs = all_logs.any?
```

If `@relievers.any?` but `!@has_logs`, set `@weekly_data = []`, `@correlation = nil`, and return early (view renders "has relievers but no logs" empty state).

Build `@weekly_data` — an array of hashes, one per week (oldest first):

```ruby
@weekly_data = build_weekly_data(all_logs, period_start)
```

Implement `build_weekly_data` as a private method:
- Iterate over each week in the period. Each week runs Monday–Sunday. The first "week" may be a partial week starting at `period_start`.
- For each week compute `week_start` (beginning of that Monday), `week_end` (end of that Sunday or Date.current if in current week).
- Count total uses (dose_log records, not puff count — one log = one use regardless of puffs) within that week window: `logs.count { |l| l.recorded_at.to_date.between?(week_start, week_end) }`.
- Derive GINA band: 0–2 → :controlled, 3–5 → :review, 6+ → :urgent.
- Build label: `week_start.strftime("%-d %b")`.
- Return array of `{ week_start:, week_end:, uses:, band:, label: }`.

**Monthly eyebrow stats (always calculated from current month, not period):**

```ruby
month_start         = Date.current.beginning_of_month
@monthly_uses       = Current.user.dose_logs
  .where(medication: @relievers)
  .where(recorded_at: month_start.beginning_of_day..)
  .count
@monthly_pill_class = monthly_pill_class(@monthly_uses)
@monthly_pill_label = monthly_pill_label(@monthly_uses)
```

Private helpers:
- `monthly_pill_class(uses)` → "eyebrow-pill--green" if ≤8, "eyebrow-pill--amber" if 9–15, "eyebrow-pill--red" if 16+
- `monthly_pill_label(uses)` → "Well controlled" / "Review recommended" / "Speak to your GP"

**Peak flow correlation:**

Query peak_flow_readings for the current user in the same period:

```ruby
pf_readings = Current.user.peak_flow_readings
  .where(recorded_at: date_range.first.beginning_of_day..date_range.last.end_of_day)
```

Build `@correlation` by joining weekly_data to peak flow readings per week:
- For each week in `@weekly_data` determine whether it is "high-use" (uses >= 3) or "low-use" (uses <= 2).
- Collect `value` from peak_flow_readings whose `recorded_at.to_date` falls in that week.
- Average values for high-use weeks → `high_avg`. Average for low-use weeks → `low_avg`.
- Set `@correlation = nil` if fewer than 2 data points total (i.e. the section stays hidden), OR if there are no high-use weeks OR no low-use weeks (can't compare).
- Otherwise: `@correlation = { high_avg: high_avg.round, low_avg: low_avg.round }`.
