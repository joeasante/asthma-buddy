  <action>
  **CSS changes in `reliever_usage.css`:**

  1. Add a `.reliever-bars-scroll` class BEFORE the `.reliever-bars` rule:
     ```css
     .reliever-bars-scroll {
       overflow-x: auto;
       -webkit-overflow-scrolling: touch;
       margin: 0 calc(-1 * var(--space-md));
       padding: 0 var(--space-md);
     }
     ```
     The negative margin + padding trick keeps the bars visually flush with the card edges while allowing the scroll container to extend edge-to-edge for a natural scroll feel.

  2. Add `min-width: 2.25rem` to the existing `.reliever-bar-col` rule. This prevents columns from collapsing below the width needed for a date label like "12 Mar". The rule already has `flex: 1` so columns will still grow on wider screens.

  3. Remove `white-space: nowrap` from `.reliever-bar-label`. The labels are short enough (e.g. "3 Mar") that they do not need nowrap, and removing it allows the column to compress more gracefully at its min-width. If a label is slightly wider than 2.25rem it will wrap to two lines rather than forcing overflow.

  **View changes in `index.html.erb`:**

  4. Wrap the existing `<div class="reliever-bars" ...>` element (line 76) in a new div:
     ```erb
     <div class="reliever-bars-scroll">
       <div class="reliever-bars" role="img" aria-label="Weekly reliever usage bar chart">
         ...existing bars content unchanged...
       </div>
     </div>
     ```
     The `.reliever-threshold` line and all `.reliever-bar-col` elements remain inside `.reliever-bars` unchanged.

  Do NOT change any other styles, HTML structure, GINA legend, correlation section, or toggle buttons.
  </action>
