## Decisions (LOCKED — honour exactly, do not revisit)

### Display format: CSS bar chart, weekly frequency
- Show **8 weekly bars** by default, with a **12-week** toggle option
- Each bar represents total reliever uses across all reliever-type medications that week
- Use pure CSS bars (not Chart.js Canvas) — simpler, accessible natively, fits the card pattern
- Period toggle uses the existing `.adherence-toggle` + `.btn-sm` + `.btn-sm--active` pattern inside a Turbo Frame

### Threshold: 3 GINA bands (not a single line)
- 0–2 uses/week → **teal** (`--teal-500`) — "controlled" (uses reliever's own colour identity, not severity-green which means preventer adherence)
- 3–5 uses/week → **amber** (`--severity-moderate`) — "review recommended"
- 6+ uses/week → **red** (`--severity-severe`) — "speak to your GP"
- A dashed horizontal threshold line sits at y=2 as a visual reference

### Multi-medication scope: combine all relievers
- Sum dose_logs across ALL `medication_type: :reliever` medications — the GINA threshold is total usage, not per-inhaler
- If the user has 2+ relievers, render a compact "Tracking: [Ventolin] [Bricanyl]" tag row using the existing `.medication-badge--reliever` class
- If the user has only 1 reliever, omit the tag row — the page title is sufficient

### Correlation: two `.dash-stat` cards, no second chart
- A "Peak Flow Correlation" section appears below the bar chart **only when** the user has peak flow readings in the same period
- Use the existing `.dash-stats` (2-column) layout: "High-use weeks avg" (zone-red coloured) vs "Low-use weeks avg" (zone-green coloured)
- One plain-language sentence below the stats: e.g. "That's a 24% difference — weeks you used your reliever more often tended to be lower-peak-flow weeks."
- "High-use" = weeks with 3+ uses; "Low-use" = weeks with 0–2 uses
- If there are no peak flow readings, or fewer than 2 data points to compare, hide the entire correlation section — never show empty stats or "N/A"

### Page header: full `.page-header` pattern with eyebrow threshold pill
- Background: teal-50 band (standard `.page-header`)
- Eyebrow stat: "This month · X uses" + `eyebrow-pill` coloured by monthly threshold status:
  - ≤8 → `eyebrow-pill--green` "Well controlled"
  - 9–15 → `eyebrow-pill--amber` "Review recommended"
  - 16+ → `eyebrow-pill--red` "Speak to your GP"
- Monthly threshold = 2/week × 4 weeks = 8 uses/month
- Icon badge: inhaler/wind SVG in `.page-header-icon`
- h1: "Reliever Usage"
- Subtitle: "Your reliever dose history"

### Empty states: two distinct states using existing `.empty-state` component
- **No reliever medications**: icon + "No reliever inhaler added" + "Add a reliever in your medication settings to start tracking usage." + `btn-primary` → `settings_medications_path`
- **Has reliever(s) but no logs yet**: icon + "No doses logged yet" + "Log a dose from the Medications page when you use your reliever." + `btn-secondary` → `settings_medications_path`

### Route and controller
- Route: `get "reliever-usage", to: "reliever_usage#index", as: :reliever_usage` (kebab-case URL, matches existing route conventions)
- Controller: `RelieverUsageController` with a single `index` action
- Data query stays inline in the controller — no new service object (logic is a simple `group_by`, not complex enough to extract)

  `link_to "Reliever Usage", reliever_usage_path, class: "dash-section-link"`
  placed directly after the "Preventer History" link
