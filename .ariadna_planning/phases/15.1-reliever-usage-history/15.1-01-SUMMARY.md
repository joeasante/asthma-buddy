## Controller Instance Variables

| Variable | Type | Description |
| -------- | ---- | ----------- |
| @weeks | Integer (8 or 12) | Selected period in weeks |
| @relievers | ActiveRecord::Relation | User's reliever medications, chronological order |
| @has_logs | Boolean | Whether any dose logs exist in the period |
| @weekly_data | Array<Hash> | One hash per week (see weekly data structure below) |
| @monthly_uses | Integer | Count of reliever dose logs in the current calendar month |
| @monthly_pill_class | String | CSS class for eyebrow pill (eyebrow-pill--green/amber/red) |
| @monthly_pill_label | String | Text label for eyebrow pill |
| @correlation | Hash or nil | Peak flow correlation data (see correlation hash below) |

## Weekly Data Structure

```ruby
{
  week_start: Date,    # First day of this week slot (may be mid-week for first slot)
  week_end:   Date,    # Last day of this week slot (Date.current for current week)
  uses:       Integer, # Count of dose log records in the window
  band:       Symbol,  # :controlled (0-2), :review (3-5), :urgent (6+)
  label:      String   # "%-d %b" formatted date, e.g. "3 Mar"
}
```

## Correlation Hash Structure

```ruby
# Present only when: pf_readings >= 2 AND high-use weeks AND low-use weeks both have readings
{ high_avg: Integer, low_avg: Integer }

# nil when insufficient data
```

## Decisions Made

1. **CSS loading pattern**: `stylesheet_link_tag "reliever_usage"` added inside the `authenticated?` block in `app/views/layouts/application.html.erb`, consistent with `health_events.css` and all other feature stylesheets. No `@import` approach used.

2. **dash-insight-text substitute**: Class does not exist in application.css. Per plan note, used `<p class="page-header-subtitle" style="margin-top: var(--space-sm);">` — closest existing small-text utility with token-based spacing.

3. **Monthly stats extracted**: `setup_monthly_stats` private method called from all three code paths (no relievers, has relievers but no logs, has logs) to avoid duplicating the month-start query and pill class computation.

4. **Turbo Frame placement**: The `<turbo-frame id="reliever-frame">` wraps only the toggle + chart + correlation section, not the empty states or page header — consistent with the adherence page pattern where the frame wraps the content that changes on period toggle.
